Linux command-line tool that opens files with your favorite editor, but also automatically uses your favorite sudo command when needed for write-permissions. This way, you won't forget to open with sudo and get that annoying "no-permissions" or "cannot save" message.

If root-permissions are required for write-access, your elevation command will ask you for the password. If they are not required, your editor command will be used alone. This is because opening all files as root may put root ownership/group status on files that don't really need it, which is also annoying.

Usage

cedit 
# ...to open a file.
	
cedit list
# ...lists current configuration (editor and elevation command)
		
cedit set editor=[your editor]
cedit set elevcmd=[your elevation-cmd]
# ...sets your favorite commands so cedit will remember what to use.

If your favorite editor isn't in the same directory as cedit or /usr/bin, then you will need to set the editor like this: 

cedit set editor=/my/path/to/myeditor

Installation

You can create a symlink to the cedit.py script yourself, or you can use the builtin 'install' command.

To install cedit in /usr/bin (for all users) run: 

sudo ./cedit.py install

To install cedit for a single user run: 

./cedit.py install user
    
# this will try to install cedit to /home/USERNAME/.local/bin
# you will need this directory to exist, and be included in your PATH variable.

If everything went well, or you manually created the symlink yourself, you should be able to run cedit like this from anywhere: 

cedit 
# with no .py extension or ./ prefix...

With no settings

If no settings are set, cedit will try to look for a few 'popular' commands to work with.

cedit will look for these editors if your favorite isn't set:

  • kate
  • gedit
  • leafpad
  • kwrite

cedit will look for these elevation commands if your favorite isn't set:

  • kdesudo
  • gksudo
  • sudo

Without any settings, if you have both kate and kdesudo installed, cedit will open your 'root-permissions' file by running: kdesudo kate If the file doesn't require root permissions then just: kate

Future

There may be plans to allow different editors to be chosen for different file-types. Right now cedit works as expected, as long as you only use one editor most often. I may add the ability to set favorite editors for favorite file-types like: 

cedit set .py=geany
